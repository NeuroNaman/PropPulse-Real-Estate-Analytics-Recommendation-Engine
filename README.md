## рҹҢҗ Overview

**PropPulse** is a production-grade real estate intelligence platform that combines machine learning price prediction, interactive analytics, and content-based property recommendations into a unified Streamlit application.

The project spans the **complete data science lifecycle** вҖ” from raw web scraping to a deployed multi-module app вҖ” targeting the Gurugram residential real estate market with sector-level granularity.

```
рҹҢҗ Scrape  в”Җв”Җв–ә  рҹ§№ Clean  в”Җв”Җв–ә  рҹ”¬ EDA  в”Җв”Җв–ә  вҡҷпёҸ Engineer  в”Җв”Җв–ә  рҹ§  Model  в”Җв”Җв–ә  рҹ“Ҡ Visualize  в”Җв”Җв–ә  рҹҡҖ Deploy
```

The platform solves three real-world problems simultaneously:

- **For buyers** вҶ’ Predict the fair price range of any property before negotiating
- **For analysts** вҶ’ Explore sector-level trends, pricing heatmaps, and BHK distributions
- **For investors** вҶ’ Discover similar high-value properties through a hybrid recommender

## рҹ”„ Data Pipeline

The raw data was scraped from a residential property listing portal and passed through a multi-stage cleaning pipeline before modeling.

### Stage 1 вҖ” Web Scraping

```
Source: Residential property listing portal (Gurugram)
Tool:   Python requests + BeautifulSoup
Output: Raw CSV вҖ” 246 projects Г— 30+ raw attributes
Fields: property_name, sector, price, built_up_area,
        bedRoom, bathroom, balcony, agePossession,
        furnishing_type, latitude, longitude, amenities_text
```

### Stage 2 вҖ” Missing Value Imputation

```
Numerical columns
в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

```

### Stage 3 вҖ” Outlier Treatment

```
Method:     IQR-based capping (Winsorization)
Target:     price, built_up_area, price_per_sqft
Transform:  log1p applied to price (target variable)
            вҶ’ Converts right-skewed distribution to ~normal
            вҶ’ Reduces influence of ultra-luxury outliers
```

### Stage 4 вҖ” Train / Test Split

```
Split ratio:   80% train  /  20% test
Strategy:      Random split  (random_state = 42)
CV Strategy:   10-Fold Cross Validation on train set
Primary metric: MAE     Secondary: RMSE, RВІ
```

## рҹҸЎ Recommender System

### Architecture

```
Given:   Selected apartment name
Goal:    Return Top-N most similar apartments

Step 1 вҖ” Build 3 Cosine Similarity Matrices
в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
  cosine_sim1  вҶҗ  TF-IDF on facility / amenity text
  cosine_sim2  вҶҗ  Cosine on 40+ price-related features
  cosine_sim3  вҶҗ  Cosine on 1,000+ geospatial features

Step 2 вҖ” Weighted Fusion
в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
  combined_score = w1 Г— sim1 + w2 Г— sim2 + w3 Г— sim3

  Default weights (user-adjustable via Streamlit sliders):
    w1 = 0.5   вҶ’  Facility / amenity similarity
    w2 = 0.8   вҶ’  Price feature similarity
    w3 = 1.0   вҶ’  Geospatial / location similarity

Step 3 вҖ” Rank and Return
в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
  Sort by combined_score descending
  Skip index 0 (self-match)
  Return: property name, similarity score, listing URL
```

### Similarity Matrix Details

| Matrix | Method | Features Used | Shape |
|--------|--------|--------------|-------|
| `cosine_sim1` | TF-IDF + Cosine | Amenity text (pool, gym, club, parkingвҖҰ) | 246 Г— 246 |
| `cosine_sim2` | Cosine similarity | 40+ price & size features | 246 Г— 246 |
| `cosine_sim3` | Cosine similarity | 1,000+ geospatial distance features | 246 Г— 246 |

### Recommender Code

```python
def recommend_properties(property_name, w1=0.5, w2=0.8, w3=1.0, top_n=5):
    # Weighted fusion of all three similarity matrices
    cosine_sim_matrix = w1 * cosine_sim1 + w2 * cosine_sim2 + w3 * cosine_sim3

    # Get similarity scores for selected property
    idx        = location_df.index.get_loc(property_name)
    sim_scores = list(enumerate(cosine_sim_matrix[idx]))

    # Sort descending, skip self (index 0)
    sorted_scores  = sorted(sim_scores, key=lambda x: x[1], reverse=True)
    top_indices    = [i[0] for i in sorted_scores[1:top_n + 1]]
    top_scores     = [round(i[1], 3) for i in sorted_scores[1:top_n + 1]]
    top_properties = location_df.index[top_indices].tolist()
    links          = (property_data
                      .set_index("PropertyName")
                      .loc[top_properties]["Link"]
                      .tolist())

    return pd.DataFrame({
        'Property Name':    top_properties,
        'Similarity Score': top_scores,
        'Link':             links
    })
```

### Current Limitations

```
1. Geographic Scope
   в””в”Җ Covers Gurugram residential market only
   в””в”Җ 246 projects may not represent all micro-markets within the city

   в””в”Җ No real-time price scraping or refresh mechanism
   в””в”Җ Model accuracy degrades naturally as market conditions evolve

   в””в”Җ App is fully stateless вҖ” no saved searches, history, or preferences

   в””в”Җ No CI/CD or automated model retraining pipeline in place
```
